react-youtube is a React component that lets you embed and control YouTube videos inside your app without manually writing <iframe> code.

Think of it as a clean wrapper around the YouTube player that works the “React way”.

Since your lectures have YouTube URLs, you use react-youtube to:

✅ show lecture videos inside your Course Details page
✅ play a video when a lecture is clicked
✅ control playback (play, pause, seek)
✅ detect when a video ends
✅ make a proper learning experience (not redirect to YouTube)

So instead of sending students away from your site → the video plays inside your platform.

useEffect(() => {
  if (initialRating) {
    setRating(initialRating);
  }
}, [initialRating]);

This runs whenever initialRating changes.
Why this is important:
Imagine: user rating comes from API, API response arrives later, component must update, Without this effect → stars would stay outdated.

uniqid

What it is:
A small utility that generates unique IDs (strings).
Why React developers use it:
Creating unique keys when rendering lists
Generating temporary IDs for items before saving to a database
Avoiding duplicate identifiers in the UI

Example use cases
✔ Adding items to a cart
✔ Creating dynamic form fields
✔ Temporary IDs for notes, messages, courses, etc. (like your course arrays)

quill (usually via ReactQuill)
What it is:
A rich text editor — like the formatting box in Google Docs.
Why React developers use it:
Let users write formatted content
Support bold, italic, headings, lists, links, images
Store HTML content from user input
This is super relevant to your project because your course descriptions already contain HTML (<h2>, <p>, etc.).
